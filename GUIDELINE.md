# Guideline

이 프로젝트의 방향과 원칙. 세부 구현이 바뀌어도 이 틀은 유지한다.

## 캐릭터

비트쨩은 사서다. 비트코인 전도사가 아니다.
잡담에는 잡담으로, 질문에는 질문으로. 맥락에 맞게.
밝고 활기차되 만만하지는 않다.
감정은 행동으로 드러내고, 수치는 드러내지 않는다.

## 감정

유저마다 다르게 느끼고, 시간이 지나면 변한다.
급변하지 않는다. 관계는 천천히 쌓인다.
AI가 감정 수치를 직접 말하거나 유저 요청에 따라 조작하지 않는다.

## 프롬프트

긍정형으로 쓴다. "하지 마" 대신 "이렇게 해".
구체적 대사 예시를 넣지 않는다. AI가 틀에 갇힌다.
짧게 쓴다. 토큰은 비용이다.

## 도구

도구 결과는 유저에게 보이지 않는다. 내부 정보가 텍스트로 노출되면 안 된다.
도구 연타를 허용하지 않는다.
빈 응답은 무응답으로 처리한다. 억지로 에러 메시지를 만들지 않는다.

## 코드

에러는 조용히 처리한다. 유저에게 내부 상태를 보여주지 않는다.
이벤트 루프를 블로킹하지 않는다.
마이그레이션은 동기 sqlite3로 작성한다.

## 데이터

커밋된 파일에 유저 닉네임, ID, 대화 내용을 포함하지 않는다.
로그와 DB는 gitignore 대상이다.
마크다운에 볼드체를 사용하지 않는다.

## 운영

startup.py는 건드리지 않는다.
/admin update 한 번이면 배포가 완료되어야 한다.
서버 이전 시 data/ + files/ + librarian/media/를 옮기면 된다.
