## v2 (2026-03-29 저녁 - 2026-03-30)

약 30시간 운영, 1,595건 대화, 에러율 13.2%.

### 좋아진 점

- **NoneType 크래시 거의 해소**: v1에서 108건 → v2에서 4건. 응답 파싱 버그 수정 효과.
- **"딴생각" 폴백 대폭 감소**: v1의 112건(5.4%) → v2의 19건(1.2%). 반복 방지 3단계가 효과적.
- **통합 search 도구 성공**: get_entry_detail 호출이 102→2로 급감. search가 충분히 풍부한 결과를 반환해서 2단계 조회가 불필요해짐.
- **웹 검색 기능**: 44건의 AI 자발 웹 검색 + 76건의 키워드 트리거. 날씨/시세/뉴스 질문에 대응 가능해짐.
- **삼행시가 인기**: 유저들이 비트코인 테마 삼행시를 많이 요청했고 잘 수행.
- **사서 역할 잘 수행한 사례**: 한 유저가 초보자 추천 → 중급 추천까지 자연스럽게 연결됨.

### 문제점

- **INVALID_ARGUMENT 여전히 120건**: 히스토리 꼬임 문제가 근본적으로 해결되지 않음. v1(191건)보다 줄었지만 여전히 최다 에러.
- **웹 검색 "잠깐만" 문제**: 날씨/시세 질문에 "잠깐만 기다려봐!" 같은 중간 응답만 내보내고 실제 데이터를 안 줌. 관리자가 4회 이상 반복 시도.
- **"주인님" 과다**: v1에서 13회 → v2에서 567회. 관리자를 공개 채널에서 "주인님!"으로 부름. 페르소나 프롬프트에서 "주인한테는 친밀하게"가 과도하게 작용.
- **학습 데이터 노출 사건**: 특정 인물에 대한 부정적 학습("~는 쓰레기")이 다른 유저에게 그대로 노출됨. 학습 데이터에 필터/검증이 없어서 발생.
- **유저 관련 학습 노출**: 한 유저에 대해 학습된 부정적 내용이 본인에게 노출되어 삭제 요청.
- **프롬프트 인젝션**: 한 유저가 특정 트리거에 미리 작성한 서버 가이드를 학습시켜 봇이 그대로 출력. 의도된 용도이긴 하나, 기억 시스템이 인젝션에 취약한 증거.
- **쓰레기 학습 축적**: 시작 시 자동 정리에서 매번 3-18건씩 삭제됨. 질문("뭐야?")이나 무의미한 문장이 learned에 저장.

## v1 (2026-03-29 오후)

약 10시간 운영, 2,078건 대화, 에러율 14.8%. 초기 폭발적 트래픽.

### 좋아진 점 (vs 없음)

- 기본 기능 동작 확인: 파일 검색(search_entries 242회), 파일 전송(send_file 88회) 작동.
- 유저들이 적극적으로 테스트 (시간당 ~208건).

### 문제점

- **히스토리 꼬임 폭발**: INVALID_ARGUMENT 191건. Gemini API가 function call 후 히스토리 구조를 깨뜨림. 177건의 히스토리 리셋 발생.
- **NoneType 크래시 108건**: 응답 파싱에서 None 체크 누락.
- **API 키 전부 소진**: 5개 키가 동시 일일 한도 초과. "딴생각에 빠져 있다" 메시지만 출력.
- **"딴생각" 112건 (5.4%)**: 에러 발생 시 유일한 폴백이 "딴생각" 메시지. 유저 불만 발생.
- **도구 미호출 문제**: 관리자가 "제발 '도구'를 사용해서 무슨 도서가 있는지 알려줘"라고 요청. 히스토리 꼬임 후 AI가 도구를 부르지 않고 추측으로 답함.
- **있는 자료를 없다고 답함**: 도서관에 있는 자료를 없다고 답해서 유저가 "거짓말쟁이"로 부름. search_entries 결과가 히스토리 꼬임으로 전달 안 됨.
- **공격적 테스트**: 한 유저의 욕설/반복 요청으로 API 키 소진 가속.
- **파일 미수신**: 봇이 전송했다고 답했으나 유저가 실제로 받지 못한 사례.
- **반복 무응답에 대한 불만**: "눈앞의 문제나 좀 해결해줄래?" — 딴생각 메시지만 반복되는 것에 대한 유저 반응.
