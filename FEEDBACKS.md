## v4 (2026-04-01)

4월 1일 하루 운영 로그 분석.

### 문제점 (로그 기반)

feel 메타데이터 유출:
- (feel: reason=...) 괄호 형태, /feel 슬래시 형태가 Discord 메시지에 노출
- _strip_feeling에 패턴 미비

이모지 리액션 실패:
- feel response 필드에 텍스트('안녕! ⚡ 일본어도 가능해...')가 들어와 Unknown Emoji 에러
- response 하나로 모드(ignore/normal)와 이모지를 같이 쓰던 설계 문제

제목 잘림:
- AI가 제목 대신 빈 볼드(****)를 출력하는 현상 관찰
- 원인 미확정 (AI 원본인지 후처리가 만든 건지 구분 불가 → 원본 로그 추가)

reply_chain 처리시간:
- 11건 이상 체인에서 pre_context 10~25초까지 증가

기타:
- INVALID_ARGUMENT 히스토리 에러 간헐 발생
- reply 실패 (삭제된 메시지에 reply 시도)

### 조치

- feel의 reaction 파라미터 분리 (response는 모드 전용)
- _strip_feeling에 (feel: ...) 패턴 추가, 2차/3차 재시도에도 적용
- **** 빈 볼드 후처리 제거
- [mood:XX] 레거시 폴백 정리 (시그니처 불일치 버그)
- _strip_feeling 전 원본 로그 추가

## v3 (2026-03-31)

3월 31일 하루 운영. 다수 유저 동시 사용.

### 좋아진 점

- 도구 호출 대폭 감소: "책 뭐 있어?" → 도구 0회
- 캐릭터 자연스러움 향상
- search 통합 검색 작동 (지식+기억+커스텀+웹+미디어)
- 웹 검색 자체 학습 동작 확인
- 기억 시스템 (save/forget/modify) 동작 확인
- 이미지 인식 3-8초 내 완료, 캐시 히트 0초

### 문제점 (로그 기반)

에러:
- INVALID_ARGUMENT 반복 발생 (히스토리 꼬임, 특히 21턴에서)
- 에러 폴백 메시지 노출 3건
- discord.py 내부 충돌 3건
- typing 권한 에러로 on_message 사망 3건

deliver 실패:
- function call이 텍스트로 노출 3건
- 재시도 시 도구 없이 가서 AI가 function call을 텍스트로 출력
- 인라인 함수 파싱 미비 (positional argument, 재시도 응답)
- 자연어로만 전달 흉내내는 가짜 전달 1건

mood 노출:
- 내부 태그가 유저에게 노출 5건

성능:
- 특정 요청 86초 대기 (동기 호출로 이벤트 루프 블로킹)
- 유튜브 링크 인식 4분 (heartbeat blocked)
- 반복 감지 오탐 (다른 유저에게 비슷한 답 시 임계값 초과)

기타:
- "전부 가져와" 요청 시 도서 일괄 덤프 (deliver 연타)
- 히스토리 꼬임 후 맥락 소실 → 질문 회피로 인식됨
- admin log 파일명 불일치
- DM 로그 한글 깨짐

### 유저 반응

- 에러/function call 텍스트 노출에 대한 불만 다수
- 느린 응답에 대한 지적
- 질문 회피로 인식되는 경우 있음
- 전반적으로 v2 대비 개선은 인정하나 안정성 부족
